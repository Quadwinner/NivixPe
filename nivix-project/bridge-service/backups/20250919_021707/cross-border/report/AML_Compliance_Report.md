## 2. AML Program Framework

### 2.1 Risk Assessment

#### 2.1.1 Customer Risk Categories
**Low Risk:**
- Verified individuals in low-risk jurisdictions
- Transaction amounts < $5,000
- Established transaction history
- No adverse media

**Medium Risk:**
- New customers with limited history
- Transaction amounts $5,000 - $25,000
- Customers from medium-risk jurisdictions
- Frequent currency changes

**High Risk:**
- Politically Exposed Persons (PEPs)
- High-risk jurisdictions (FATF grey/black list)
- Transaction amounts > $25,000
- Unusual transaction patterns
- Cash-intensive businesses

#### 2.1.2 Geographic Risk Assessment
**Low Risk Jurisdictions:**
- United States, Canada, United Kingdom
- European Union member states
- Australia, New Zealand, Singapore

**Medium Risk Jurisdictions:**
- India, Brazil, Mexico
- South Africa, Turkey, Thailand

**High Risk Jurisdictions:**
- FATF blacklist countries
- Countries with weak AML controls
- Sanctioned jurisdictions

#### 2.1.3 Product Risk Assessment
**Cross-Border Payments:** Medium-High Risk
- Rationale: International transfers are inherently higher risk
- Mitigation: Enhanced monitoring, transaction limits, KYC requirements

### 2.2 Customer Due Diligence (CDD)

#### 2.2.1 Identity Verification
**Tier 0 (Limited Access):**
- Email verification
- Phone number verification (SMS OTP)
- Transaction limit: $1,000

**Tier 1 (Standard Access):**
- Full name, date of birth, address
- Government-issued ID (passport, driver's license, national ID)
- Selfie verification (liveness check)
- Transaction limit: $10,000

**Tier 2 (Enhanced Access):**
- Tier 1 requirements
- Proof of address (utility bill, bank statement < 3 months)
- Source of funds declaration
- Enhanced screening
- Transaction limit: $50,000

**Tier 3 (Business Access):**
- Business registration documents
- Beneficial ownership information (UBO)
- Financial statements
- Business purpose documentation
- Transaction limit: Unlimited (subject to monitoring)

#### 2.2.2 Enhanced Due Diligence (EDD)
Applied to high-risk customers:
- Source of wealth verification
- Purpose of relationship
- Expected transaction activity
- Ongoing monitoring (monthly reviews)
- Senior management approval

### 2.3 Transaction Monitoring

#### 2.3.1 Automated Monitoring Rules

**Rule 1: Structuring Detection**
- Trigger: Multiple transactions just below reporting threshold
- Threshold: 3+ transactions of $9,000-$9,999 within 24 hours
- Action: Flag for manual review

**Rule 2: Rapid Movement**
- Trigger: Funds received and immediately sent to different jurisdiction
- Threshold: < 1 hour between receipt and send
- Action: Flag for manual review

**Rule 3: High-Risk Jurisdiction**
- Trigger: Transaction to/from FATF high-risk jurisdiction
- Threshold: Any amount
- Action: Enhanced review, possible block

**Rule 4: Velocity Check**
- Trigger: Unusual increase in transaction frequency or amount
- Threshold: 300% increase over 30-day average
- Action: Flag for manual review

**Rule 5: Round Amount**
- Trigger: Multiple round-number transactions
- Threshold: 5+ transactions ending in 000 within 7 days
- Action: Flag for manual review

**Rule 6: Dormant Account Reactivation**
- Trigger: Account inactive > 90 days suddenly active with large transaction
- Threshold: Transaction > $5,000
- Action: Enhanced verification required

#### 2.3.2 Real-Time Screening
- **Sanctions Lists:** OFAC SDN, UN, EU, UK HMT
- **PEP Lists:** World-Check, Dow Jones
- **Adverse Media:** Automated news screening
- **Update Frequency:** Daily
- **Matching Algorithm:** Fuzzy matching with 85% threshold

#### 2.3.3 Manual Review Process
1. Alert generated by automated system
2. Assigned to compliance analyst within 1 hour
3. Investigation (review transaction history, customer profile, external data)
4. Decision: Approve, Request Information, File SAR, Block Account
5. Documentation in case management system
6. Senior review for SAR filings

### 2.4 Suspicious Activity Reporting (SAR)

#### 2.4.1 SAR Triggers
- Transactions with no apparent lawful purpose
- Customer refuses to provide information
- Structuring or smurfing patterns
- Transactions involving high-risk jurisdictions
- Unusual use of business accounts
- Transactions inconsistent with customer profile

#### 2.4.2 SAR Filing Process
1. Compliance analyst identifies suspicious activity
2. Gather supporting documentation
3. Complete SAR form (FinCEN Form 111)
4. Senior compliance officer review
5. File with FinCEN within 30 days
6. Maintain confidentiality (no customer notification)
7. Document in secure system

#### 6.2.2 Sanctions Screening (Real-Time)
```javascript
async function screenTransaction(sender, recipient, amount, currency) {
    // Check sender against sanctions lists
    const senderCheck = await sanctionsAPI.check(sender);
    if (senderCheck.isMatch) {
        return { blocked: true, reason: 'Sender on sanctions list' };
    }
    
    // Check recipient against sanctions lists
    const recipientCheck = await sanctionsAPI.check(recipient);
    if (recipientCheck.isMatch) {
        return { blocked: true, reason: 'Recipient on sanctions list' };
    }
    
    // Check jurisdiction risk
    const jurisdictionRisk = await checkJurisdictionRisk(sender, recipient);
    if (jurisdictionRisk === 'HIGH') {
        return { flagged: true, reason: 'High-risk jurisdiction' };
    }
    
    return { approved: true };
}
```

#### 6.2.3 Velocity Checks
```javascript
async function checkVelocity(userId, amount, currency) {
    const last24h = await getTransactionSum(userId, '24h');
    const last7d = await getTransactionSum(userId, '7d');
    const last30d = await getTransactionSum(userId, '30d');
    
    const userTier = await getUserKYCTier(userId);
    const limits = TRANSACTION_LIMITS[userTier];
    
    if (last24h + amount > limits.daily) {
        return { blocked: true, reason: 'Daily limit exceeded' };
    }
    
    if (last30d + amount > limits.monthly) {
        return { blocked: true, reason: 'Monthly limit exceeded' };
    }
    
    // Check for unusual velocity
    const avg30d = last30d / 30;
    if (amount > avg30d * 5) {
        return { flagged: true, reason: 'Unusual transaction size' };
    }
    
    return { approved: true };
}
```
